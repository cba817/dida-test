# 网站管理后台开发计划

## 一、概述

为「空间增长系统」网站开发一个完整的管理后台，实现 Logo/站点名称、诊断痛点、套餐内容、成功案例的在线管理功能。

## 二、需求清单

| # | 模块 | 功能 |
|---|------|------|
| 1 | Logo及站点设置 | 上传Logo图片、修改网站名称、修改网站描述文字 |
| 2 | 痛点内容管理 | 对三种驱动类型（产品/渠道/用户）的痛点进行增删改查 |
| 3 | 套餐内容管理 | 对三个套餐的价格、功能列表、推荐阈值进行增删改查 |
| 4 | 案例管理 | 对成功案例进行增删改查，含图片上传功能 |

## 三、当前状态分析

### 3.1 现有架构
- **前端**：HTML + CSS + Vanilla JS，无框架
- **后端**：Express.js（端口 3001）
- **存储**：JSON 文件（`server/db/leads.json`）
- **痛点/套餐/案例**：全部硬编码在 `js/state.js`、`js/app.js`、`server/routes/api.js` 中

### 3.2 核心问题
- 痛点数据硬编码在 `js/state.js`（`config.driverPains`）
- 套餐数据硬编码在 `js/app.js`（`renderPackages()`）
- 案例数据硬编码在 `server/routes/api.js`（`CASES` 数组）
- 无管理员认证机制
- 无文件上传支持

## 六、API设计

### 6.1 前台公开API（无需认证）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/site` | 获取站点设置 |
| GET | `/api/pains?driverId=PRODUCT` | 获取驱动痛点 |
| GET | `/api/packages` | 获取套餐列表 |
| GET | `/api/cases` | 获取案例列表（改为读取db） |

### 6.2 管理后台API（需JWT认证）

**认证：**
- `POST /api/admin/login` — 登录获取token
- `GET /api/admin/check` — 验证token有效性

**站点设置：**
- `GET /api/admin/site` — 获取站点设置
- `PUT /api/admin/site` — 更新站点名称/描述
- `POST /api/admin/site/logo` — 上传Logo（multipart）

**痛点管理：**
- `GET /api/admin/pains` — 获取全部痛点
- `POST /api/admin/pains` — 新增痛点
- `PUT /api/admin/pains/:painId` — 编辑痛点
- `DELETE /api/admin/pains/:painId` — 删除痛点

**套餐管理：**
- `GET /api/admin/packages` — 获取全部套餐
- `POST /api/admin/packages` — 新增套餐
- `PUT /api/admin/packages/:pkgId` — 编辑套餐
- `DELETE /api/admin/packages/:pkgId` — 删除套餐

**案例管理：**
- `GET /api/admin/cases` — 获取全部案例
- `POST /api/admin/cases` — 新增案例
- `PUT /api/admin/cases/:caseId` — 编辑案例
- `DELETE /api/admin/cases/:caseId` — 删除案例
- `POST /api/admin/cases/:caseId/image` — 上传案例图片

## 八、关键技术决策

1. **认证方式**：JWT，无状态、无需session存储
2. **文件上传**：multer，支持图片大小/类型校验
3. **数据fallback**：前台保留硬编码数据作为后备，API不可用时不影响使用
4. **诊断建议引擎**：保持原样不纳入管理后台（需求未覆盖）
5. **图片存储**：本地 `server/uploads/` 目录，被.gitignore排除
6. **管理员账号**：默认 admin / admin123，存储在 config.js

## 九、验证方式

1. 访问 `http://localhost:3001/admin` — 显示登录页
2. 输入 admin/admin123 登录 — 进入管理面板
3. 分别测试4个模块的增删改查功能
4. 访问前台页面 — 检查数据是否同步更新
5. 关闭后端服务 — 前台仍能使用本地fallback数据正常展示
